## Given

A sublinear time bounded *cellular tree automaton* (CTA) $M$ and an input string $x$.

## Problem

Does $M$ accept $x$?

## Definitions

See Ibarra, Jiang, and Chang <a href="#ref-1" class="reference-citation">[1]</a>.

## Status

$\P$-hard (Ibarra, Jiang, and Chang <a href="#ref-1" class="reference-citation">[1]</a>).

## Remarks

A *trellis automaton* can accept a $\P$-complete language (Ibarra and Kim <a href="#ref-2" class="reference-citation">[2]</a>). This in turn can be simulated by a *linear iterative array* that in turn can be simulated by a cellular tree automaton. See <a href="#ref-1" class="reference-citation">[1]</a> and <a href="#ref-2" class="reference-citation">[2]</a> for definitions and further details.

A sublinear time one-way bounded CTA can also accept a $\P$-complete language <a href="#ref-1" class="reference-citation">[1]</a>. In a *one-way* CTA, communication between vertices is bottom-up. Every language accepted by an oneway CTA can be accepted by a deterministic Turing machine in $(\log n)^2 / \log \log n$ space, where $n$ is the size of the input <a href="#ref-1" class="reference-citation">[1]</a>.
