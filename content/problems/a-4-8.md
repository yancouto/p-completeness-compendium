## Given

An ordered list of $n$ jobs $\\{J_1, \ldots, J_n\\}$, a positive integer execution time $T(J_i)$ for each job, and a non-preemptive schedule $L$. The jobs are to be scheduled on two identical processors.

## Problem

Is the final offset produced by the list scheduling algorithm nonzero? The *final offset* is the difference in the total execution time of the two processors.

## Status

$\P$-complete (Helmbold and Mayr <a href="#ref-1" class="reference-citation">[1]</a>).

## Remarks

The problem is in $\NC$ if the job times are small, that is $n^{O(1)}$. $\NC$ algorithms for scheduling problems with either intree or outtree precedence constraints are known (Helmbold and Mayr <a href="#ref-2" class="reference-citation">[2]</a><a href="#ref-1" class="reference-citation">[1]</a>). Also see [HPS]({{< relref "./a-4-9.md" >}}) for additional scheduling results. See Mayr <a href="#ref-3" class="reference-citation">[3]</a> for $\NC$ approximation algorithms. Sunder and He show that List Scheduling is $\P$-complete <a href="#ref-4" class="reference-citation">[4]</a>. The differences between *List Scheduling* and GLS are that all jobs have unit time and the number of processors is an input to the problem. That is, it is not fixed at two as in GLS.

Sunder and He prove their result via a reduction from [HPS]({{< relref "./a-4-9.md" >}}). They also show that List Scheduling is in $\NC$ if the precedence constraints are an interval order. An *interval order* is a partial order $G = (V, E)$, where $V$ is a set of intervals on the real line and $([l_1, r_1], [l_2, r_2]) \in E$ if and only if $r_1 < l_2$. Their algorithm runs in $O((\log n)^2)$ time using $n^5$ processors on a PRIORITY CRCW-PRAM. Another version of the algorithm runs in $O((\log n)^3)$ time using $n^4$ processors on the same model <a href="#ref-4" class="reference-citation">[4]</a>.
