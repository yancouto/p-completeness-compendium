## Given

Two $n$-bit positive integers $a$ and $b$.

## Problem

Compute the greatest common divisor (gcd) of $a$ and $b$, denoted $\gcd(a, b)$.

## Status

Open (von zur Gathen <a href="#ref-3" class="reference-citation">[3]</a>).

## Remarks

For $n^\text{th}$ degree polynomials $p, q \in \Q[x]$, computing $\gcd(p, q)$ is in $\NC^2$ via an $\NC^1$ reduction to *Determinant* (Cook and Sethi <a href="#ref-1" class="reference-citation">[1]</a>, Borodin, Cook, and Pippenger <a href="#ref-2" class="reference-citation">[2]</a>). IntegerGCD is $\NC^1$ reducible to [SV2]({{< relref "./b-7-2.md" >}}) <a href="#ref-3" class="reference-citation">[3]</a>. Kannan, Miller, and Rudolph give a sublinear CRCW-PRAM algorithm for computing the gcd of two $n$-bit integers <a href="#ref-4" class="reference-citation">[4]</a>. Their algorithm requires $O(n \log \log n / \log n)$ time and $n^2 (\log n)^2$ processors. Chor and Goldreich give a CRCW-PRAM algorithm for the same problem with time bound $O(n / \log n)$ and processor bound $n^{1+\epsilon}$, for any positive constant $\epsilon$ <a href="#ref-5" class="reference-citation">[5]</a>.

Sorenson presents an algorithm called the $k$-ary gcd algorithm <a href="#ref-6" class="reference-citation">[6]</a><a href="#ref-7" class="reference-citation">[7]</a> whose sequential version is practical and whose parallel version has bounds that match those given in <a href="#ref-5" class="reference-citation">[5]</a>. Also see (Deng <a href="#ref-8" class="reference-citation">[8]</a>, Lin-Kriz and Pan <a href="#ref-9" class="reference-citation">[9]</a>, Shallcross, Pan, and Lin-Kriz <a href="#ref-10" class="reference-citation">[10]</a>) for additional research on IntegerGCD and related problems.
